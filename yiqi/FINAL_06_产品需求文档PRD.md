# 1. 产品概述

## 1.1 产品名称（暂定）

**待定**，建议方向：
- 「夜书」：你的夜间日记本
- 「梦记」：简单直接
- 「潜台词」：潜意识的台词
- 「第二人生」：呼应"梦是第二人生"的理念
- 「梦语」：梦在说话
- 「做梦的人」：带一点文艺感

> **白话**：产品名需要后续专门做一轮品牌讨论，最好结合 logo 一起定。

## 1.2 一句话定位

**「一款用 AI 帮你翻译梦境、看见潜意识的情绪觉察工具。」**

## 1.3 核心价值主张（Slogan 候选）

- **「你不是一个人，你是一群你。」**
- **「每一个梦，都是一个版本的你。」**
- **「把梦里的情绪，翻译成人话。」**
- **「梦是你的第二人生。」**

## 1.4 产品差异化（为什么用户要选你？）

市场上已有的梦境类产品大致分两类：
- **玄学解梦类**（如周公解梦 APP）：停留在"梦到蛇代表什么"的字典式解读，陈旧、缺乏个性化
- **心理日记类**（如小日常、Moodfit）：偏通用情绪记录，不聚焦梦境

**本产品的差异化核心**：
1. **AI 驱动的深度个性化分析**，不是查字典
2. **心理学底子 + 当代抽象玩梗调性**，有温度也有记忆点
3. **"状态底色 + 人格类型"双维度**，兼顾"当下情绪"和"性格画像"
4. **数据可视化档案馆**，让用户看到自己潜意识的长期画像
5. **猫咪统一 IP**，打造独特视觉资产

# 2. 目标用户与使用场景

## 2.1 目标用户画像

**核心用户**：15-30 岁年轻人，男女比例均衡
- 高频用户：对自我探索感兴趣、有情绪觉察需求的年轻女性（18-28 岁）
- 次高频：对 MBTI、星座、心理测试感兴趣的人群
- 潜在用户：有睡眠或情绪困扰的人群（但产品不定位为医疗工具）

**用户城市分布**：一二线城市为主，三四线城市为辅
**用户心理特征**：
- 喜欢自我剖析、爱做 MBTI 测试
- 在小红书、抖音、豆瓣活跃
- 喜欢抽象玩梗的表达风格
- 愿意为"情绪价值"付费

## 2.2 核心使用场景

**场景一：清晨记录**
用户刚醒来，还记得昨晚的梦——打开 APP，用语音或文字快速记下，AI 生成分析报告，用户边刷牙边看，获得一天的"情绪预警"。

**场景二：午休或夜间复盘**
用户白天突然想起前一晚的梦，或者夜里被梦惊醒，打开 APP 记录，获得即时的情绪释放。

**场景三：月度/季度回顾**
用户在档案馆查看本月梦境类型分布、情绪曲线、主导人格——获得对自己近期状态的宏观感知。

**场景四：分享传播**
用户觉得某次的分析报告"太准了"，截图发小红书/朋友圈/闺蜜群——产品获得免费传播。

# 4. 主要用户流程

## 4.1 首次使用流程（新用户）

```
下载 APP
  ↓
打开 APP - 开屏页（3 秒过渡）
  ↓
首次欢迎引导（3 页滑动介绍）
  ↓
登录 / 注册（手机号 / 微信 / Apple ID）
  ↓
个人信息初始化（昵称、生日-可选、性别-可选）
  ↓
隐私与使用条款确认（重点说明"不是诊断"）
  ↓
首页（引导用户点击"+"记录第一个梦）
```

## 4.2 记录梦境的完整流程（核心流程）

```
首页点击"+"按钮
  ↓
进入记录页 - 选择记录方式
  ↓
【方式 A】文字输入           【方式 B】语音输入
  ↓                              ↓
用户打字                     用户按住说话
  ↓                              ↓
点击"完成"                   松手自动转文字
  ↓                              ↓
    ↓────────────────────────────↓
              ↓
AI 检查内容完整性（检查维度见下）
  ↓
缺信息？
  ├─ 是 → AI 礼貌追问（最多 2 轮）→ 用户回答 or 跳过
  └─ 否 → 直接生成报告
  ↓
选择梦的情绪色卡（7 类，可由 AI 预选）
  ↓
（可选）添加标题、备注
  ↓
确认生成报告
  ↓
加载动画（3-8 秒，展示"解梦师正在分析中"）
  ↓
进入报告页（5 个模块）
  ↓
用户可：保存 / 分享 / 返回首页
```

### AI 内容完整性检查的 4 个维度

AI 会自动检测用户输入是否覆盖了以下 4 个维度，不足时会追问：

| 维度 | 简写 | 举例 |
|---|---|---|
| 情绪 | emotion | 愤怒、害怕、平静、开心… |
| 行为 | act | 被追赶、坠落、吵架、迷路… |
| 事物 | sth | 水、房子、车、牙齿掉… |
| 人物 | sb | 亲人、前任、陌生人、敌人… |

**追问规则**：
- 最多追问 2 轮
- 每轮最多 1 个问题
- 用户回答"没有"或"不记得"时不强求，直接生成报告
- 追问话术要温和，例如："那梦里你有什么情绪感受？比如害怕、平静、开心？"

## 4.3 查看历史报告流程

```
首页下方 Tab 切换到"梦境"
  ↓
看到历史记录列表（含时间、标题、类型标签、情绪色块）
  ↓
点击某条记录
  ↓
查看完整分析报告
  ↓
可：重新分析 / 编辑备注 / 分享 / 删除
```

## 4.4 查看档案馆流程

```
首页下方 Tab 切换到"档案馆"
  ↓
看到 3 个可视化图表（情绪曲线 / 类型饼图 / 词云）
  ↓
点击任一图表可放大查看、切换时间范围
  ↓
点击词云里的词 → 跳转到含该词的所有梦境列表
```

# 5. 页面结构与详细设计

## 5.1 整体导航结构

APP 采用**底部 Tab 导航**（像微信那种），共 3 个 Tab：

```
[🏠 首页] [📖 梦境] [👤 我的]
```

其中"档案馆"作为独立入口放在**首页** or **我的**里（待定，建议放在首页的二级页面，因为数据可视化是核心卖点）。

> **白话**：Tab 就是底部的那几个切换按钮，用户点一下就切到对应页面。

## 5.2 各页面详细设计

---

### 页面 1：首页（核心）

**功能**：引导用户记录梦境 + 快速入口

**页面元素**：
- 顶部：问候语（根据时间变化：早上"做了个什么样的梦？" / 晚上"今天辛苦了，睡前想记点什么吗？"）
- 中部：**巨大的"+"按钮**（核心 CTA，用户最主要的操作）
- "+" 旁边提示文字："点击记录梦境 · 语音或文字"
- 下方：**档案馆卡片入口**（显示本月记录数、最近主导人格，点击进入档案馆）
- 底部：最近 1-2 条历史梦境缩略（方便快速回看）

**设计调性**：
- 背景：星空感、渐变色、夜间氛围
- "+"按钮：有微动效（脉动 / 光晕），吸引用户点击

**关键交互**：
- 点 "+" → 进入记录页
- 点档案馆卡片 → 进入档案馆
- 点历史缩略 → 进入那次的报告

---

### 页面 2：记录梦境页（核心）

**功能**：输入梦境内容 + AI 追问补充

**页面元素**：
- 顶部：返回按钮 + "记录梦境"标题
- 中部：**文字/语音切换按钮**（默认语音）
- 语音模式：**按住说话的大圆按钮**（像微信语音）
- 文字模式：输入框 + 键盘
- 转文字区域（上方）：显示语音转出的文字
- AI 追问区域：聊天气泡样式
- 底部：**"完成"按钮**（点击触发 AI 分析）

**关键交互**：
- 语音按钮：长按说话 / 松手停止 / 实时转文字
- 用户可手动修改转出的文字
- 追问以聊天气泡形式出现，用户回答后气泡更新
- 追问最多 2 轮，完成后 "完成" 按钮变亮

**用户状态**：
- 未开始：按钮置顶显眼
- 输入中：实时字数统计（可选显示）
- 追问中：用户可"跳过此问题"
- 加载中：显示"解梦师正在分析中…"

### 页面 3：梦境分析报告页（产品灵魂）

**功能**：展示 AI 生成的完整分析报告

**页面结构**（严格按此顺序）：

#### 区块 1：状态 & 人格卡片
- 🌡 你当前的状态底色：[状态词]
- 🖼 [猫咪插画 - 2D 动漫风]
- 💬 [人格 slogan]
- 🎭 你当前的潜意识人格倾向：[人格名 + emoji]
- 🎨 配色：[配色]
- 📖 人格画像：[画像]
- 📊 匹配度：[XX%] + 说明

**视觉要求**：整个区块做成一张"人格卡片"，背景色用该人格对应的配色，有卡片边框、阴影，具有收藏感和分享价值。

#### 区块 2：梦境深度解读
- 📖 标题：梦境深度解读
- 展开多个维度小标题（AI 根据梦境选 2-5 个）：
  - 情绪基调
  - 核心冲突
  - 关键象征
  - 梦中自我状态
  - 关键人物
  - 现实对照
  - 总结提醒
- 每个维度下 1-3 句解读文字

#### 区块 3：六维评分
- 📊 标题：六维评分
- 六个维度（固定）：压力指数、情绪稳定度、内心冲突、睡眠指数、心理放松度、安全感指数
- 每个维度：分数（0-100）+ 进度条 + 一句话解读
- 底部：小字提示（"得分仅反映关联程度，非严重程度"）

#### 区块 4：解语花
- 🌹 标题：这朵解语花送你
- 一段 3-5 句的温暖文字
- 视觉：加重、居中、花朵装饰

#### 区块 5：友情提示
- ⚠️ 固定免责声明文案
- 视觉：字小、色淡、置底

**底部操作栏**：
- [保存到档案馆] [分享] [重新生成] [返回]

**关键交互**：
- 整个报告支持下拉滚动查看
- 每个模块可单独截图分享
- 整份报告可生成长图一键分享

# 6. AI 分析引擎规则

## 6.1 AI 的角色设定

AI 化身为"**解梦师**"，调性是：**科学底子 + 心理咨询师的温度 + 当代年轻人的抽象玩梗**。

详细的提示词见另一份文档《梦境分析 AI · 系统提示词 V1.1》。

## 6.2 分析流程

```
用户输入梦境
  ↓
内容完整性检查（4 维度）
  ↓
不完整 → 追问（最多 2 轮）
  ↓
调用大模型 API（Claude / GPT / 豆包等）
  ↓
根据提示词输出 5 模块报告
  ↓
报告渲染到 UI
```

> **白话**："调用大模型 API"就是 APP 把用户的梦发给 AI 服务器，AI 服务器返回分析结果。这需要付费，按调用次数算钱，约每次 0.1-0.5 元人民币。

## 6.3 AI 输出的关键字段

一次完整的报告包含以下字段（供开发参考）：

| 字段 | 类型 | 说明 |
|---|---|---|
| state_color | string | 状态底色词 |
| personality_name | string | 人格名 |
| personality_emoji | string | 人格 emoji |
| personality_slogan | string | Slogan（从 21 人格库固定取） |
| personality_color | string | 配色 |
| personality_portrait | string | 人格画像 |
| illustration_desc | string | 猫咪插画描述 |
| match_score | number | 匹配度（60-95） |
| match_description | string | 匹配度说明 |
| deep_analysis | array | 梦境深度解读（2-5 个维度对象） |
| six_scores | object | 六维评分（6 个数值 + 解读） |
| healing_words | string | 解语花文字 |
| disclaimer | string | 友情提示（固定文案） |

## 6.4 异常处理

- **用户梦境太短（少于 10 字）**：追问"可以多说一点吗？比如梦里你在做什么？"
- **用户输入非梦境内容**：AI 礼貌说明"这里只解析梦境内容哦，其他想聊的可以去我们的……"
- **用户梦境涉及自伤/自杀**：不做娱乐化分析，温和提示并附上心理援助热线
- **AI 接口失败**：显示"网络有点小波动，稍后再试"，记录失败次数

# 10. 隐私、安全与伦理边界

## 10.1 数据隐私

- **数据加密**：所有梦境内容加密存储
- **本地优先**：尽可能支持本地存储，云端仅用于 AI 分析
- **用户控制**：用户可随时导出或删除自己的所有数据
- **不卖数据**：明确承诺不将用户数据出售给第三方

## 10.2 伦理边界（非常重要）

这是产品的**红线**，任何情况下都不能越过：

### 🚫 严禁越界行为

- **不做医疗诊断**：不使用"抑郁症、焦虑症、PTSD（医学意义）"等诊断词
- **不做玄学预言**：不预测吉凶、不谈前世今生、不算命
- **不绝对化判断**：永远用"可能、似乎、像是"这类软化词
- **不利用恐惧营销**：不用"你的梦很危险"这种话吓用户付费

### ⚠️ 特殊情况处理

**情况 1：用户梦境或回答中出现自伤、自杀信号**
- 立刻切换到严肃模式，不做娱乐化分析
- 显示温和提示："读到你的梦，我有点担心你。如果你最近情绪特别低落，可以和信任的人或专业人士聊聊。"
- 附上心理援助热线（北京心理危机研究与干预中心：010-82951332）

**情况 2：涉及未成年人（用户标注生日 < 18 岁）**
- 简化报告内容，避免过度挖掘
- 不生成涉及暗恋、情感创伤等成人话题的深度解读
- 加强敏感词过滤

**情况 3：涉及他人隐私（用户描述里出现真实姓名等）**
- 前端提示："为保护他人隐私，建议不要写具体姓名"
- 后端自动脱敏处理

## 10.3 首次使用的用户教育

在新用户欢迎引导页，清晰说明：
- 这是**情绪觉察**工具，不是医疗诊断
- 梦境分析**仅供参考**，不要影响重大现实决策
- 数据如何被使用、如何保护
- 用户可以随时删除所有数据

# 11. 技术实现建议（给未来开发用）

> **说明**：这部分写给技术人员看，你作为产品负责人了解就行，不必深究。

## 11.1 推荐技术栈

### 一期 MVP（最小可用产品）建议方案

**方案 A：H5 网页版 / 小程序（推荐新手创始人）**
- 前端：React / Vue
- 后端：Node.js / Python
- AI 接口：Claude API / OpenAI API / 国内豆包/Kimi API
- 数据库：MySQL / MongoDB
- 部署：阿里云 / 腾讯云
- 优势：开发周期 2-3 个月，成本 5-15 万
- 劣势：体验略逊原生 APP

**方案 B：原生 APP（iOS + 安卓）**
- iOS：Swift
- 安卓：Kotlin
- 或跨平台：React Native / Flutter
- 优势：体验最好
- 劣势：开发周期 4-6 个月，成本 20-40 万

**方案 C：无代码工具（完全没有预算）**
- 用 FlutterFlow / Bubble / Glide 搭建
- AI 接口用 Zapier / n8n 连接
- 优势：不用写代码，几周搞定
- 劣势：扩展性差，高级功能受限

> **建议**：新手创始人优先选方案 A（小程序+H5），验证市场后再考虑原生 APP。

## 11.2 关键技术挑战

1. **语音识别**：集成讯飞 / 百度语音识别 SDK
2. **AI 调用成本控制**：加缓存、限制用户调用频率、选择便宜的模型
3. **敏感内容过滤**：接入内容安全 API（阿里云绿网、腾讯内容安全）
4. **数据可视化**：ECharts / Chart.js 基础够用

## 11.3 AI 成本估算

- 单次分析：约 0.1-0.5 元（取决于用哪个模型）
- 免费用户每日 3 次：每月人均成本约 9-45 元
- 因此：**免费用户需要 5-10% 转化为付费才能打平**
- 建议：前 3 个月烧钱做种子用户，6 个月内达到收支平衡
