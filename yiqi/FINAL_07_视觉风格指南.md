# 1. 风格定位与气质

## 1.1 核心气质

**「深夜里的一盏暖灯」**

- **深夜** = 暗色底，神秘、梦境感
- **暖灯** = 温暖色点缀，治愈、有温度
- **一盏** = 极简、克制，不过度装饰

## 1.2 调性关键词

**主关键词**：神秘、治愈、温暖、文艺、精致
**辅关键词**：有设计感、小红书化、年轻化、抽象、留白

## 1.3 避免的感觉

- ❌ 冰冷科技感（硅谷 SaaS 风）
- ❌ 少女粉甜腻（妆品 APP 风）
- ❌ 玄学神秘主义（塔罗占卜 APP 风）
- ❌ 学术心理学（专业量表 APP 风）
- ❌ 国际化模板感（通用英文 APP 套皮）

## 1.4 同类产品视觉参考

- **网易云音乐 · 深色模式**：暗色 + 大图 + 卡片式布局
- **小宇宙播客**：简约中带文艺感
- **苹果音乐 · 暗色**：深色底 + 高对比色点缀
- **Notion 暗色模式**：克制的层次感
- **小红书的高质量内容卡片**：可截图分享的构图

# 2. 色彩系统

## 2.1 核心色板

### 底色系（背景用）

| 名称 | 色值 | 用途 |
|---|---|---|
| 深夜蓝紫 | `#1A1B2E` | APP 主背景（暗色模式） |
| 梦境紫 | `#2D1F3D` | 次级背景、卡片底 |
| 墨夜黑 | `#0F0F1A` | 最深处的阴影 |
| 雾夜灰 | `#3D3550` | 分隔线、边框 |

### 主色（内容、重点用）

| 名称 | 色值 | 用途 |
|---|---|---|
| 暖灯橙 | `#FFD4A3` | 核心强调色（主按钮、重点数据、高光） |
| 奶油米 | `#F5E6D3` | 正文文字（替代白色，更柔和） |
| 月光紫 | `#C9A7D4` | 次级强调色（标签、装饰） |
| 雾蓝 | `#7B9DB0` | 中性强调色（辅助信息） |

### 文字色（暗色模式）

| 名称 | 色值 | 用途 |
|---|---|---|
| 主文字 | `#F5E6D3` | 标题、正文 |
| 次文字 | `#B8A8C9` | 副标题、说明文字 |
| 辅助文字 | `#6B6580` | 时间戳、次要信息 |
| 禁用文字 | `#4A4458` | 禁用状态 |

### 浅色模式色板（日间模式）

| 名称 | 色值 | 用途 |
|---|---|---|
| 奶油白底 | `#FAF6F0` | APP 主背景 |
| 白卡片 | `#FFFFFF` | 卡片底 |
| 暖灰 | `#E8E0D4` | 分隔线 |
| 深紫字 | `#2D1F3D` | 主文字（替代纯黑） |
| 次级字 | `#6B6580` | 次文字 |

> **设计理念**：浅色模式不是纯白，而是奶油米底——延续暖灯气质。主色依然是暖灯橙，只是对比方向反过来。

### 语义色（状态提示用）

| 用途 | 色值 | 说明 |
|---|---|---|
| 成功 | `#8BB88C` | 温和的抹茶绿 |
| 警告 | `#E8A87C` | 温和的暖橘 |
| 错误 | `#C97B8A` | 温和的玫红（不用刺眼红） |
| 信息 | `#7B9DB0` | 雾蓝 |

## 2.2 人格卡片专属配色（21 个）

每个人格有自己的主辅色组合（已在《状态底色与人格图鉴 V3》中定义）。人格卡片的背景会使用该人格的配色，让每次报告都有独特的视觉身份。

**示例**：
- 快乐小狗：蜜桃橙 `#FFB088` + 奶白 `#FFF5E8`
- 深情种：酒红 `#8B3A47` + 旧玫瑰 `#C48B93`
- 松弛大师：雾蓝 `#8BA8C4` + 浅绿 `#B8D4C2`

## 2.3 颜色使用原则

- **60-30-10 法则**：60% 底色、30% 主色、10% 强调色
- **暗色模式下，永远不用纯白（#FFFFFF）**，改用奶油米 `#F5E6D3`——更温暖
- **浅色模式下，永远不用纯黑（#000000）**，改用深紫 `#2D1F3D`——更文艺
- **色彩承担情绪而非装饰**：一个页面颜色太多 = 情绪混乱

# 3. 字体系统

## 3.1 字体选择

### 中文字体

| 场景 | 字体 | 备选 | 说明 |
|---|---|---|---|
| **标题 · 文艺感** | 思源宋体 / Source Han Serif | 方正兰亭宋、仓耳今楷 | 用于报告页大标题、状态底色词 |
| **标题 · 力量感** | 思源黑体 / Source Han Sans | 苹方、阿里汉仪 | 用于人格名、数据 |
| **正文** | 苹方 / PingFang SC | 思源黑体 | 报告正文、菜单、按钮 |
| **Slogan · 手写感** | 思源宋体斜体 | 霞鹜文楷 | 人格 slogan、解语花 |

### 英文字体

| 场景 | 字体 | 说明 |
|---|---|---|
| **英文装饰** | Playfair Display / Cormorant Garamond | 优雅衬线，报告页副标签（如 "YOUR MOOD TODAY"） |
| **英文数据** | Inter / SF Pro Display | 清爽无衬线，数字、英文 UI |

## 3.2 字号规范

### 报告页

| 元素 | 字号 | 字重 | 字体 |
|---|---|---|---|
| 状态底色词（大标题） | 32px | 500 | 思源黑体 |
| 人格名 | 22px | 500 | 思源黑体 |
| 英文小标题 | 11px | 400 | Playfair Display（字间距 2） |
| 匹配度数字 | 13px | 500 | Inter |
| Slogan | 20px | 500 | 思源宋体斜体 |
| 解语花 | 13px | 400 | 思源宋体 |
| 正文解读 | 14px | 400 | 苹方 |
| 免责声明 | 11px | 400 | 苹方 |

### 通用 UI

| 元素 | 字号 | 字重 |
|---|---|---|
| 页面大标题 | 24px | 500 |
| 卡片标题 | 16px | 500 |
| 正文 | 14px | 400 |
| 辅助文字 | 12px | 400 |
| 最小字号 | 11px | 400 |

## 3.3 字体使用原则

- **一个页面最多 3 种字体**（比如思源宋体 + 苹方 + Playfair）
- **大小对比要明显**：标题 vs 正文至少差 8px，否则层次混乱
- **斜体用得克制**：只在 slogan 和解语花使用，不要滥用
- **英文小标题全大写 + 字间距**：增加杂志感（如 `YOUR MOOD TODAY`）
- **行高**：正文 line-height = 1.7（呼吸感），标题 1.3

# 4. 版式规范

## 4.1 留白与间距

**原则**：宁可少，不可挤。留白就是呼吸，就是设计。

| 场景 | 间距 |
|---|---|
| 安全边距（页面左右） | 20-24px |
| 卡片内边距 | 24px（上下左右） |
| 卡片之间 | 16px |
| 段落之间 | 20px |
| 文字段落内部 | 8-12px |

## 4.2 圆角规范

| 场景 | 圆角 |
|---|---|
| 大卡片（人格卡） | 24px |
| 中卡片（内容卡） | 16px |
| 小卡片（标签、按钮） | 12px |
| 输入框 | 12px |
| 圆形按钮 | 50%（正圆） |

## 4.3 网格系统

移动端以 **4px 为最小单位**，所有间距、尺寸都是 4 的倍数（4、8、12、16、20、24、32、40、48...）。这样页面所有元素有呼应关系。

## 4.4 报告页版式原则

- **每个模块一张卡片**，模块之间留有间距，方便用户截图分享
- **卡片里的信息有清晰层次**：英文小标题 → 中文大标题 → 正文 → 补充信息
- **大量留白**：每张卡片内部顶部、底部都留 24-32px 空
- **左对齐为主**，居中用得克制（只用在特别强调的一行字）

# 5. 组件规范

## 5.1 按钮

### 主要按钮（Primary）

- 背景：暖灯橙 `#FFD4A3`
- 文字：深夜蓝紫 `#1A1B2E`
- 字重：500
- 高度：48px
- 圆角：24px（胶囊形状）
- 内边距：左右 32px

### 次要按钮（Secondary）

- 背景：透明
- 边框：0.5px 奶油米 `#F5E6D3` 半透明
- 文字：奶油米
- 高度：48px
- 圆角：24px

### 文字按钮（Text Only）

- 文字：暖灯橙
- 无背景、无边框
- 字重：400
- 可选：下划线装饰

## 5.2 输入框

- 背景：`#2D1F3D`（比底色稍亮一点）
- 边框：无（聚焦时 0.5px 暖灯橙）
- 文字：奶油米
- 占位符：辅助文字色
- 高度：48px
- 圆角：12px
- 内边距：16px

## 5.3 卡片

### 人格卡片（核心组件）

- 背景：使用该人格的配色（渐变）
- 圆角：24px
- 边框：0.5px 半透明雾夜灰
- 内边距：24px
- 最小高度：480px（方便截图分享）

### 内容卡片

- 背景：梦境紫 `#2D1F3D`
- 圆角：16px
- 边框：0.5px 雾夜灰 `#3D3550`
- 内边距：20px

## 5.4 标签（Tag / Chip）

- 高度：28px
- 圆角：14px（胶囊）
- 内边距：左右 12px
- 字号：12px
- 情绪色系背景：半透明（0.2 不透明度）
- 文字：对应色系的亮色

## 5.5 进度条

- 轨道：雾夜灰 `#3D3550`
- 填充：暖灯橙 `#FFD4A3`
- 高度：6px
- 圆角：3px
- 动画：生成报告时从 0 增长到目标值（1.5 秒）

## 5.6 分隔线

- 颜色：雾夜灰 `#3D3550`
- 粗细：0.5px（永远不是 1px，太重）
- 透明度：0.6

# 6. 插画风格规范

## 6.1 核心规则

**所有插画统一使用猫咪形象**，作为产品的视觉 IP。

## 6.2 插画风格

- **2D 动漫风 + 简约线条**
- **线条粗细**：主线 1.5px，辅线 0.8px
- **颜色**：配合当前人格的配色，不脱离色板
- **表情和姿态是灵魂**：21 个人格靠表情和动作区分，不靠物种
- **背景**：留空或极简（一个月亮、一个窗户、一朵云）

## 6.3 插画尺寸

- **报告页主插画**：正方形，边长 200-240px
- **档案馆小插画**：圆形，直径 60-80px
- **引导页大插画**：长方形，400 × 300px

## 6.4 插画创作方向（未来执行时用）

### 前期（MVP）方案

选项 1：**AI 生成**（成本低，速度快）
- 用 Midjourney / Stable Diffusion / 国内工具
- 设定统一 prompt 模板，保持风格一致
- 每个人格出 3-5 张候选，人工筛选

选项 2：**找插画师定制**（贵，但质量可控）
- 预算：每张 300-1000 元
- 21 个人格共需约 1-2 万元

选项 3：**购买现成素材 + 改**（折中）
- 在 Pinterest / 涂鸦王国 / 小红书找参考
- 请设计师按参考微调

**建议**：MVP 阶段先用 AI 生成，如果产品有热度再投资定制。

# 7. 动效与交互

## 7.1 动效原则

- **克制**：不要抖、不要弹、不要炫技
- **柔和**：所有动效用 ease-in-out 曲线，不用 linear 或 bounce
- **短暂**：大部分动效 200-400ms，超过 500ms 会让人感觉卡顿
- **有意义**：动效为了传达信息或反馈，不是装饰

## 7.2 关键动效

| 场景 | 动效 | 时长 |
|---|---|---|
| 页面切换 | 淡入淡出 | 300ms |
| 按钮点击 | 缩放 0.98 | 150ms |
| 卡片加载 | 上移 8px + 淡入 | 400ms |
| 生成报告 | 进度条增长 + 打字机 | 3-8 秒 |
| 首页"+"按钮 | 轻微脉动 | 2 秒循环 |
| 猫咪插画 | 尾巴轻摆 / 眨眼 | 循环（可选） |

## 7.3 加载动画

生成分析报告时，页面应该：
- 显示一只坐着打盹的小猫（loading 动画）
- 下方文字："解梦师正在解读你的梦境…"
- 进度提示可以有：
  - 正在识别梦境情绪…
  - 正在匹配人格类型…
  - 正在撰写分析报告…

这个等待时间其实是**价值塑造的关键**——让用户觉得 AI 真的在认真"思考"。

# 9. 落地建议

## 9.1 设计工具推荐

- **Figma**：做 UI 设计和原型（主流工具，免费够用）
- **Procreate / Photoshop**：画插画
- **Midjourney**：AI 辅助出视觉参考

## 9.2 设计师招聘建议

如果未来要找设计师合作，告诉他们：

> "我在做一款梦境记录 + AI 分析的 APP，产品气质是『深夜里的一盏暖灯』——暗色底、暖色调点缀，有神秘感也有治愈感。视觉风格参考网易云暗色 + 小宇宙播客 + 小红书的精致卡片感。核心 IP 是猫咪插画。"

把这份《视觉风格指南》给他，他一周内就能出初稿。

## 9.3 核心验证方式

**做完任何一个页面设计后，问自己三个问题**：

1. 有"深夜感"吗？（暗色、神秘、静谧）
2. 有"暖灯感"吗？（温度、治愈、不冰冷）
3. 能截图发小红书吗？（有设计感、有记忆点、能传播）

三个都"是" → 过关
任何一个"否" → 重新调整
